ACProtect(Stolen Code) 脱壳

2019-12-12

查壳

拉进OD，设置异常

Shift+F9来到最后一次异常处，数据窗口跟随堆栈窗口的SE处理程序，然后在SE处理程序下内存访问断点

Shift+F9运行两次，来到如图的位置

在下面几行的位置，F2下断点，然后F9两次

再F4运行到retn处，到了retn处，删除所有断点

RUN跟踪，跟踪条件 push ebp，跟踪步入，找到被偷取的代码

跟踪完成后，停在被偷走的代码处，右键->二进制->二进制复制

ALT+M打开内存窗口，在代码段401000 F2下断，F9来到OEP

这里就是OEP，代码被偷掉了一部分，所以开头变成了push esi

将偷的代码粘贴回来，然后在push ebp处新建EIP

然后DUMP，ImportREC修复有很多无效指针

用1.6版本的ImportREC的跟踪级别3可以修复，但是选中OD中的进程会卡死，所以另开进程，然后在REC中选择新开的进程

将无效的用级别3修复，一次选中几个无效指针修复，选中太多也会卡死，修复完成后就没有无效指针，转储即可
