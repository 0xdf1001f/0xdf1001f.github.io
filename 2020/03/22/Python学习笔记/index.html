0x00 使用入门

问答环节

Python是什么？

Python是一种跨平台的计算机程序设计语言，是一门多种用途的编程语言，时常在扮演脚本语言的角色。一般来说，Python可定义为面向对象的脚本语言

Python的优点

软件质量

在很大程度上，Python更注重可读性，一致性和软件质量。Python代码的设计致力于可读性，因此具备了比传统脚本语言更优秀的可重用性和可维护性。即时代码不是亲手所写，Python的一致性也保证了其代码易于理解。

提高开发者效率

相较于C，C++，JAVA等编译/静态类型语言，Python的开发者效率提升了数倍，Python代码的大小只有C++或JAVA代码的1/5~1/3，这就意味着可以录入更少的代码，调试更少的代码并在之后维护更少的代码。

程序的可移植性

绝大多数Python程序不做任何改变即可在所有的主流计算机平台上运行，例如在Linux和Winodws之间移植Python代码，只需要简单的复制粘贴即可

标准库的支持

Python内置了众多预编译并可移植的功能模块，这些功能模块叫做标准库（standard library）。标准库支持一系列应用级的编程任务，涵盖了从字符模式到网络脚本编程匹配等方面。Python可通过自行开发的库或众多第三方应用支持软件进行扩展。

组件集成

Python脚本可通过灵活的集成机制轻松地与应用程序的其他部分进行通信。

Python的缺点

在目前现有的实现方式下，与C和C++这类编译语言相比，Python的执行速度还不够快。Python的实现方式是将源代码的语句编译（或者说是转换）为字节码的形式，再将字节码解释出来。字节码是一种与平台无关的格式，具有可移植性。然而，Python没有将代码编译成底层的二进制代码（Intel芯片的指令），一些Python程序会比C这样的完全编译语言慢一些。

Python如何运行程序

Python解释器简介

解释器是一种让其他程序运行起来的程序。当你编写了一段Python程序，Python解释器将读取程序，并按照其中的命令执行，得出结果。解释器是代码与机器的计算机硬件之间的软件逻辑层

程序执行

字节码编译

当程序执行时，Python会先将源代码编译成所谓的字节码形式。编译是一个简单的翻译步骤，而且字节码是底层的，与平台无关的表现形式。概括的说，Python通过把每一条源语句分解成单一步骤来将这些源语句翻译成一组字节码指令。这些字节码可以提高运行速度，比起文件中的原始源代码语句，字节码的运行速度会快得多。（.pyc就是编译过的.py源代码），如果Python无法在本地写入字节码，程序仍然可以正常工作，字节码会在内存中生成并在运行结束后丢弃。

Python虚拟机（PVM）

一旦程序编译成字节码（或字节码已经从.pyc中载入），字节码会发送到Python虚拟机（Python Virtual Machine，简写PVM）中执行。从技术上讲，PVM才是解释器的最后一步

冻结二进制文件

冻结二进制文件可以将程序的字节码、PVM以及任何程序所需要的Python支持文件捆绑在一起形成一个单独的文件包。结果会得到一个单独的可执行的二进制程序（Windows下的.exe），及时在没有Python环境的系统上，依然能够运行

如今又三种系统能够生成冻结二进制文件：py2exe（Winodws）、PyInstall（Linux及UNIX）以及freeze

习题

1.什么是Python解释器？

Python解释器是运行Python程序的程序

2.什么是源代码

源代码是为程序所写的语句，包含了文件文件（通常为.py）的文本

3.什么是字节码

字节码是Python将.py中的源代码编译后得到的底层形式，通常保存在.pyc文件中

4.什么是PVM

PVM是Python的运行引擎，编译后的字节码发送到PVM中执行

5.CPython、Jython、IronPython有什么不同

CPython是Python语言的标准实现。Jython和IronPython分别是Python程序的JAVA和.NET实现，他们都是Python的编译器替代实现。
